首先有两个文件（如果没有则需创建）：
一个已完成任务文件，一个未完成任务文件。

实现一个任务结构体：
```cpp    
struct event{
    string kind;//任务所属类别
    string name;//任务名
    string start,end;
    int Crucial;//重要性
    int time;//预计需要完成的时间
};
```
程序开始时，应该从已完成任务文件和未完成任务文件中，读入两个不同的数组中。


有如下功能：

写一个面板，等待用户进行如下操作：

List todo：列出当前可执行的所有任务数量，然后先输出一行类别，然后输出完成当前类别总预计时间，然后把类别下的所有任务名按重要性排序输出（任务名，重要性，预计时间）

List done：列出已经执行完的所有任务：输出格式为
- 第一行输出完成任务数量和总时间：
- 先输出类别：对于每个类别，需要有单独的文件，如果文件存在内容，在类别下面输出内容
- 然后输出类别下面对于每一条任务：输出任务名，开始时间结束时间，花费时间。

add： 要求用户输入任务，kind,name,string,Crucial,但是crucial没必要每次都输入，未输入的设成默认值default。

run：分三种执行方式
- random：按任务重要性带权随机一个任务。
- seq：选权重最大的任务。
- select：挑选任务名，完成当前任务。
- 这三种执行方式选定任务后，输出任务信息：同时弹出一个倒计时窗口，提醒用户剩余时间，允许用户超出预计时间，等待用户停止任务并记录。
- 在弹出倒计时窗口之后，需要到创建好的类别的文件夹下生成一个任务名的md文件（命名为 任务名.md）。
- 对于完成的任务要从未完成文件中删除，并插入到已完成结构体中。

程序结束后，再次开启能够通过读入文件回到上述使用状态
